# AI Assistant — Bedrock Chatbot

A domain-specialized conversational AI assistant for **Azercell** (Azerbaijan's largest mobile operator), built on **AWS Bedrock** with a **FastAPI** backend and a branded **Streamlit** frontend. Responses are grounded in Azercell domain context and generated by **Claude 3.7 Sonnet**, with an **Amazon Titan** fallback path. Fully containerized and deployed to **AWS EC2** via CI/CD.

---

## Architecture

```
┌──────────────────┐      HTTP       ┌──────────────────┐      ┌──────────────────┐
│  Streamlit        │  ───────────▶   │   FastAPI         │ ───▶ │   AWS Bedrock     │
│  Frontend         │  /chat /health  │   Backend         │      │   Claude 3.7      │
│  (Azercell UI)    │  ◀───────────   │                   │ ◀─── │   (Titan fallback)│
└──────────────────┘                 └──────────────────┘      └──────────────────┘
```

The backend wraps each user message in an Azercell domain-context prompt, calls Bedrock, and returns the generated answer. Model selection (Claude vs. Titan) is resolved automatically from the configured model ID.

---

## Features

- **AWS Bedrock integration** — Claude 3.7 Sonnet for response generation, with an Amazon Titan code path as a fallback.
- **Domain grounding** — answers are conditioned on Azercell company/service context for relevant, on-brand responses.
- **FastAPI backend** — clean REST API with health checks and a (simulated) streaming endpoint.
- **Branded Streamlit UI** — custom CSS, Azercell color scheme, chat history, and a responsive layout.
- **Containerized** — backend and frontend run as separate Docker services via docker-compose.
- **CI/CD** — GitHub Actions workflow validates dependencies and Docker configuration on every push.
- **Cloud-deployed** — runs on AWS EC2 with public access.

## Getting Started

### Prerequisites
- Python 3.9+
- AWS account with **Bedrock access** to Claude
- (Optional) Docker & docker-compose

### 1. Configure environment

Create a `.env` file in the project root:

```env
AWS_ACCESS_KEY_ID=your_access_key
AWS_SECRET_ACCESS_KEY=your_secret_key
AWS_REGION=us-east-1
CLAUDE_MODEL_ID=anthropic.claude-3-7-sonnet-20250219-v1:0
BACKEND_HOST=0.0.0.0
BACKEND_PORT=8000
```

### 2. Run with Docker

```bash
docker-compose up --build
```

| Service   | URL                     |
|-----------|-------------------------|
| Frontend  | http://localhost:8501   |
| Backend   | http://localhost:8000   |

### 3. Run locally (without Docker)

```bash
pip install -r requirements.txt

# backend
cd backend && python main.py

# frontend (separate terminal)
cd frontend && streamlit run app.py
```

## API Endpoints

| Method | Endpoint       | Description |
|--------|----------------|-------------|
| `GET`  | `/`            | Root status |
| `GET`  | `/health`      | Health check (verifies Bedrock connectivity) |
| `POST` | `/chat`        | Generate a chat response |
| `POST` | `/chat/stream` | Streamed chat response |

**Example request:**

```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "What 5G services does Azercell offer?"}'
```
